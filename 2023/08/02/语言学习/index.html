print和sys.stdout.write区别

print 和 sys.stdout.write 都是 Python 中用于输出内容的方法，但它们有一些不同之处：

  1. print 是一个内置函数，而 sys.stdout.write 是 sys 模块中的方法。因此，print 可以直接在代码中使用，而 sys.stdout.write 需要先导入 sys 模块才能使用。
  2. print 可以同时输出多个值，自动以空格分隔，而 sys.stdout.write 只能输出一个字符串，不能自动添加分隔符。 例子 print(12,32,43,54);
  3. print 会在输出内容的末尾添加一个换行符，而 sys.stdout.write 不会自动添加换行符，需要手动添加 \n 才能实现换行。

有下划线的方法有什么不同

方法名以双下划线 __ 开头和结尾的方法是 Python 中的特殊方法，也称为魔术方法或者魔术属性。这些 特殊方法在类中有特殊的用途

==它们会在特定情况下自动调用，而不需要我们手动调用。==

例如：

  1. __init__: 在创建对象时自动调用，用于对象的初始化。

  4. __sub__: 当使用 - 运算符进行减法操作时自动调用，实现对象的减法运算。
  5. __mul__: 当使用 * 运算符进行乘法操作时自动调用，实现对象的乘法运算。
  6. __eq__: 当使用 == 运算符进行相等比较时自动调用，实现对象的相等比较。

JavaScript this 关键字

面向对象语言中 this 表示当前对象的一个引用。

但在 JavaScript 中 this 不是固定不变的，它会随着执行环境的改变而改变。

  • 在方法中，this 表示该方法所属的对象。
  • 如果单独使用，this 表示全局对象。
  • 在函数中，this 表示全局对象。
  • 在函数中，在严格模式下，this 是未定义的(undefined)。
  • 在事件中，this 表示接收事件的元素。
  • 类似 call() 和 apply() 方法可以将 this 引用到任何对象。

JavaScript let 和 const

使用 var 关键字声明的变量不具备块级作用域的特性，它在 {} 外依然能被访问到。

let 声明的变量只在 let 命令所在的代码块 {} 内有效，在 {} 之外不能访问。

const 关键字

const定义常量与使用let 定义的变量相似：

  • 二者都是块级作用域
  • 都不能和它所在作用域内的其他变量或函数拥有相同的名称

两者还有以下两点区别：

  • const声明的常量必须初始化，而let声明的变量不用
  • const 定义常量的值不能通过再赋值修改，也不能再次声明。而 let 定义的变量值可以修改。
