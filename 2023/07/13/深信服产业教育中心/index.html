端口转发/映射

端口转发是一种网络配置技术，用于将传入的网络请求从一个网络地址和端口号转发到另一个网络地址和端口号上。它通常用于解决网络中的访问限制或者实现某些特定的网络服务。

在一个网络中，每个设备或者服务都可以通过唯一的IP地址和端口号来标识。当其他设备或者服务想要访问某个特定的设备或服务时，它们可以通过指定目标设备的IP地址和端口号来发起请求。

端口转发允许网络管理员在网络设备（通常是路由器或者防火墙）上设置规则，当收到特定端口的请求时，将这些请求转发到预定义的目标设备和端口上。这样，外部设备或者网络可以通过访问转发后的端口，实际上访问到了目标设备或者服务。

端口映射

端口映射（Port Mapping）和端口转发（Port Forwarding）是相同的概念，有时也称为端口转换。它是一种网络技术，用于将外部网络请求映射到内部网络的特定设备或服务上。

当网络中存在一个路由器或防火墙时，它通常有一个公共IP地址，而内部网络中的设备有各自的私有IP地址。当外部设备想要访问内部网络中的某个设备或服务时，就需要通过端口映射将请求从公共IP地址和端口号转发到内部设备的私有IP地址和端口号上。

信号技术

信号有多种类型，以下是一些常见的信号技术：

  1. 模拟信号：模拟信号是连续变化的信号，它可以取任意数值。模拟信号的表示方式可以是连续的电压、电流或其他物理量。常见的模拟信号技术包括模拟电视信号、模拟音频信号等。
  2. 数字信号：数字信号是离散的信号，它只能取特定的数值。数字信号通过对连续模拟信号进行采样和量化得到。数字信号可以表示为二进制的位流，其中每个位代表一个离散的数值。数字信号在计算机和通信系统中广泛使用。
  3. 调制解调：调制解调是一种将模拟信号转换为数字信号（调制）或将数字信号转换为模拟信号（解调）的技术。调制是通过改变模拟信号的某些特性（如振幅、频率、相位）来表示数字信号。解调则是将调制后的信号恢复为原始的数字信号。
  4. 数字调制：数字调制是一种将数字信号转换为模拟信号的技术。常见的数字调制技术包括脉冲编码调制（PCM）、正交频分复用（OFDM）、相移键控调制（PSK）、频移键控调制（FSK）等。
  5. 光纤通信：光纤通信是利用光纤作为传输介质进行信息传输的技术。它利用光的特性进行信号传输，光信号在光纤中通过光的折射和反射来传输信息。

未经调制的信号可以是模拟信号或数字信号

十三课网络协议层基础

IP协议

举例：假设我们有一台电脑（主机A）和一台服务器（主机B），它们通过一个以太网局域网进行通信。在OSI模型的帮助下，它们可以进行以下通信过程：

  1. 应用层：主机A的用户在Web浏览器中输入了一个网址，比如[http://www.example.com。浏览器将这个URL转换成HTTP请求，==浏览器在应用层添加HTTP头部信息==，然后将请求传递给下一层

  2. 传输层：传输层负责将应用层的数据进行分段，并为每个数据段添加序列号和校验和，以确保数据在传输过程中的可靠性和完整性。==主机A的传输层将HTTP请求数据进行分段，并添加相关信息==然后传递给下一层。

  3. 网络层：网络层负责==路由选择和寻址==，决定数据包的转发路径。==主机A的网络层将传输层的数据进行封装，并添加源IP地址和目标IP地址，==然后将数据包传递给下一层。

  1. 数据链路层：数据链路层负责将网络层传来的数据包转换成帧，并在帧头和帧尾添加相关控制信息，进行物理地址（MAC地址）寻址。==主机A的数据链路层将网络层的数据包进行封装，并添加MAC地址信息，==然后传递给下一层。

  2. 物理层：物理层负责将数据帧转换成电信号或光信号，并在物理媒介上传输。==主机A的物理层将数据帧转换成电信号，并通过以太网局域网将数据发送到主机B==。

  3. 物理层：主机B的物理层接收到来自主机A的电信号，并将其转换成数据帧。

  4. 数据链路层：主机B的数据链路层接收到来自物理层的数据帧，并解析出MAC地址信息，进行寻址和控制信息的检查。

  5. 网络层：主机B的网络层接收到来自数据链路层的数据包，并解析出源IP地址和目标IP地址，根据路由表决定数据包的转发路径。

  6. 传输层：主机B的传输层接收到来自网络层的数据，并进行校验和和序列号的检查，将分段的数据重新组合成HTTP请求。

  7. 应用层：主机B的应用层接收到来自传输层的HTTP请求，并根据请求内容在服务器上找到相应的网页内容。

==现实中的例子==

  1. 应用层：浏览器使用HTTP协议发送GET请求到www.baidu.com，。

  2. 传输层：浏览器将HTTP请求分割为数据包，并添加TCP头部信息，确保数据可靠传输。

  3. 网络层：浏览器使用DNS服务将www.baidu.com解析为对应的IP地址（例如：220.181.38.148），并将数据包传递给路由器。

  4. 数据链路层：数据包通过物理网络传输到目标主机。在每一跳，数据链路层使用目标主机的MAC地址来寻址，并将数据包传递给下一个路由器。

  5. 物理层：数据包通过以太网电缆或Wi-Fi信号等物理媒介传输到目标主机www.baidu.com。

  6. 应用层：==数据==

    • 作用：为用户提供网络服务，处理用户应用程序的通信细节。
    • 提供设备：服务器、客户端计算机。
    • 例子：使用HTTP协议传输网页数据、使用SMTP协议发送电子邮件。

  7. 传输层：==分段==

    • 作用：提供端到端的数据传输，确保数据在不同主机之间可靠传输。
    • 提供设备：主机的操作系统内核。
    • 向下层传输的信息：传输层数据包，包含源端口号和目标端口号。
    • 例子：使用TCP协议确保可靠的数据传输，使用UDP协议进行实时数据传输，如音频和视频流。

  8. 网络层：==数据包==

    • 作用：负责数据包的路由和寻址，将数据包从源主机传输到目标主机。
    • 提供设备：路由器。
    • 向下层传输的信息：网络层数据包，包含源IP地址和目标IP地址。
    • 例子：使用IP协议进行全球范围内的数据路由，将==数据包==从一个网络传输到另一个网络。

  9. 链路层：==数据帧==

    • 作用：负责将数据包从一个==节点传输到相邻节点==，确保数据在局域网中可靠传输。
    • 提供设备：交换机、网桥、网卡。
    • 向下层传输的信息：链路层数据帧，包含源MAC地址和目标MAC地址。
    • 例子：使用以太网协议在局域网内传输数据帧。
  • 应用层：对应OSI模型的应用层、表示层、会话层。
  • 传输层：对应OSI模型的传输层。
  • 网络层：对应OSI模型的网络层。
  • 链路层：对应OSI模型的数据链路层、物理层。

路由原理

路由 基础

​ 路由: 是指从源结点到目的节点的传输路线,

路由器分类:

  1. 家用路由器和企业级路由器：家用路由器主要用于家庭或小型办公网络，通常拥有较少的网络接口和较简单的功能，适用于小范围的局域网。而企业级路由器通常用于大型企业、数据中心等复杂的网络环境，拥有更多的接口和更强大的功能，能够处理大量的数据流量和复杂的路由选择。
  2. 无线路由器和有线路由器：无线路由器支持Wi-Fi无线连接，可以为无线设备提供网络连接，如手机、平板电脑等。有线路由器则主要通过以太网接口提供有线网络连接，适用于需要稳定、高速连接的设备
  3. 企业级分布式路由器和核心路由器：企业级分布式路由器通常部署在分支机构或办公室，用于连接到总部网络。核心路由器是网络中的中心设备，用于处理网络中的所有数据流量

假设有三个局域网，分别是 A、B 和 C。每个局域网都有一个路由器连接到互联网。路由器 R1 连接到局域网 A 和 B，路由器 R2 连接到局域网 B 和 C。

  2. 当局域网 B 中的路由器收到来自局域网 A 的路由信息时，它将更新自己的路由表，知道到达局域网 A 的路径是通过 R1，跳数为 2。类似地，当局域网 B 中的路由器收到来自局域网 C 的路由信息时，它将更新自己的路由表，知道到达局域网 C 的路径是通过 R2，跳数为 1。
  3. 现在，局域网 A 中的路由器希望发送数据到局域网 C。由于 A 和 C 不直接相连，它会将数据发送给局域网 B 的路由器。在这个过程中，路由器 R1 就充当了中转站，它将收到的数据包转发给局域网 C。跳楼是2
  4. 类似地，当局域网 C 中的路由器希望发送数据到局域网 A，数据将通过路由器 R1，R2 中转。

传输层概述

传输层与网络层区别

IP协议提供主机之间的逻辑通信，负责将数据包从源主机发送到目标主机

传输层协议提供进程之间的逻辑通信，负责将数据从源进程发送到目标进程，确保数据的可靠传输。

TCP和UDP区别

TCP（传输控制协议）是一种面向连接的、可靠的协议。它提供数据的可靠传输，确保数据按顺序到达目的地，并且可以进行数据的重传，以保证数据的完整性。TCP适用于对数据传输可靠性要求较高的场景，例如网页浏览、电子邮件发送等。

UDP（用户数据报协议）是一种无连接的、不可靠的协议。它不进行数据的可靠传输和重传，而是直接将数据发送出去，不关心是否能够成功到达目的地。UDP适用于对数据传输实时性要求较高的场景，例如音频和视频传输、在线游戏等。

在实际网络通信中，不同的应用程序和服务可以选择使用TCP或UDP作为传输层协议，取决于其对数据传输的要求。有些应用程序可能同时使用TCP和UDP，以满足不同数据传输需求。例如，在网络游戏中，TCP用于传输游戏状态和其他非实时数据，而UDP用于传输玩家的实时位置和动作。
